## 二、关键技术决定

### 1. 类型设计(`src/data/types.ts`)
- `Beat` 设计为 union(不再像 v0 那样什么字段都可能有的 `any`):
  ```ts
  type Beat =
    | { speaker: string; line: string; narration?: false }
    | { narration: true; line: string }
    | { choices: Choice[] }
    | { gotoChapter: string }
    | { gotoTrust: true }
    | { gotoEnding: true };
  ```
- `StoryPage.tsx` 内对 beat 用 `"key" in beat` 类型守卫窄化(TS 严格模式必需,不要回退到 `as any`)
- `EndingId = "ash" | "sealed" | "living"` 是字面量联合;`ENDINGS` 用 `Record<EndingId, Ending>`,新增结局必须同步扩 `EndingId`

### 2. 路由与状态
- 不引入 react-router——v0 就是 switch 状态机,保留这种简洁
- `PageKey` 单独抽到 `src/lib/routes.ts`,避免 pages 互相依赖 `App.tsx`
- `BottomNav` 的 `NavKey` 是 `PageKey` 子集(`story | clue | map | gallery`),`onNav` 签名兼容 `gotoPage`

### 3. localStorage
- `STORAGE_KEY = "loremender:huatuo:v1"` 与 v0 一致——已存档的玩家不会丢进度
- `loadState()` 合并 `defaultState()` 与 JSON,容忍字段缺失/解析失败(catch 静默返回默认值)

### 4. main.tsx 入口
- `StrictMode` 保留(Vite 模板默认)
- 删除模板的 `App.tsx`/`App.css`/`index.css`/`assets/`

### 5. index.html
- 已替换 Vite 默认模板,使用原 HTML 的 meta(viewport-fit=cover、maximum-scale=1、user-scalable=no、theme-color)
- Google Fonts link 直接放在 `<head>`,未做 self-host

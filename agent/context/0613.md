## 二、修复的问题(均带客观验证)

1. **首页下方空白 / 高度不自适应**
   - 根因:`CoverPage` 根节点 `style={{position:"relative"}}` 覆盖了 `.page { position:absolute; inset:0 }`,使封面按内容高度收缩。只有首页有此 inline 覆盖。
   - 修复:删除该 inline 覆盖 → 撑满全屏。
2. **整屏可上下拖动/溢出**(真机移动浏览器)
   - 根因:`.stage` 用 `min-height:100vh` 未降级 dvh;`html/body` `overflow:visible`。地址栏收起时 `100vh` 比可视区大。
   - 修复:全局 `100vh → 100dvh`(带 vh 回退);`@media(max-width:480px)` 锁 `html/body/.stage` `overflow:hidden`。验证:移动视口 `canScroll=false`。
3. **对话框有时空白**
   - 根因:`gotoChapter`/`gotoTrust`/`gotoEnding`/`gotoChapter:"clue"` 这类纯跳转 beat 无 `line`,`StoryPage` 走到时渲染空 `DialogueBox` 等用户点。
   - 修复:抽出 `runTransition()`,加 `useEffect([beat])` "到达即自动跳转";渲染分支对 `isTransition` 返回 `null`。验证:从 ch1 一路点到自动进入「第二章」,无空框。
4. **Toast 溢出右屏**
   - 根因:`.toast` 的 `animation: fadeInDown ... both` 结束态 `transform:none` 覆盖了居中用的 `translateX(-50%)`,左边缘卡在 50% 处右移。
   - 修复:专用 `@keyframes toastIn`(全程保留 `translate(-50%, …)`);`position:fixed→absolute`(限定在手机框内)、`white-space:normal` 允许换行。验证:`left56/right334`(视口 390)。

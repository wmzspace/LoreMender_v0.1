## 1. 文档概述

本文件描述互动叙事游戏 **《典故修补者 · 青囊残卷》** 的产品定位、世界观、玩法机制、内容结构、各界面规格、信息架构、数据与存档、UI/UX 规范、技术架构与路线图，作为开发、设计与内容创作的统一依据。

- **产品代号**：LoreMender（典故修补者）
- **首个副本**：华佗 · 青囊残卷（The Lore Mender · 千古遗恨）
- **形态**：移动端优先的网页互动小说（visual novel + 抉择 + 多结局）

---

## 2. 产品定位与愿景

### 2.1 一句话定位
玩家穿越进"被遗忘的典故世界"，附身关键人物，在**无法改写命运**的限制下，**修补那些被历史掩埋的遗憾**。

### 2.2 设计愿景
- 以中国历史典故为母题，用"互动叙事 + 情感修复"承载文化表达。
- 强调**遗憾的不可逆**：玩家不能让华佗不死，只能决定"医道是否断绝"。
- 每个副本是一段典故的"修补任务"，通过抉择导向不同结局（结局分级体现"修补程度"）。

### 2.3 体验目标
- 沉浸的古风视觉与文字氛围（水墨、宣纸、朱印、烛火、金箔）。
- 轻量、可一夜通关的单副本叙事（约 5–10 分钟）。
- 多结局 + 结局图鉴，鼓励复玩探索。

## 4. 核心玩法与机制

### 4.1 玩法循环
1. 进入副本 → 逐段阅读剧情（轻触推进打字机文本）。
2. 在关键节点做**抉择**（影响 flag / 推进剧情）。
3. 通过底部导航查阅**线索板**、**副本进程**、**结局图鉴**。
4. 在终章做出**唯一绑定决策**（青囊何归：托付于人 / 毁去残卷）。
5. 触发对应**结局**，解锁结局图鉴，可重新选择或回到青史空间。

### 4.2 机制要点
- **线性章节 + 章内抉择**：4 个剧情章节顺序推进，章内抉择主要塑造氛围与记录 flag。
- **唯一决策点**：最终结局由"青囊何归"界面的选择决定（见 §6.4），其余抉择不直接决定结局。
- **线索收集（可选）**：线索板为侧栏参考，收集 ≥3 条触发"领悟"文案；不阻塞主线。
- **进度续读**：离开剧情页（切到线索/进程/图鉴）再返回，**从原位续读**（按章持久化阅读位置）。
- **存档**：自动存档于浏览器 localStorage，可继续 / 重新选择 / 重置。

## 5. 世界观与设定（青史空间）

| 设定 | 含义 |
|---|---|
| **青史空间** | 所有流传下来的典故都会在此形成投影。 |
| **历史污染** | 误传、遗忘与篡改，使人物的真实情感被掩埋。 |
| **修补者** | 玩家身份。不是上帝，不能随意改写生死。 |
| **修补原则** | 不能强行逆天改命，只能让被遮蔽的价值重新被看见。 |

> 设定在"典籍设定（World）"界面以四张宣纸卡呈现。

| id | 名称 | 批注（主题） |
|---|---|---|

> 线索均围绕"知识如何流传"的母题；收集 ≥3 条触发领悟文案。

### 6.4 终章决策：青囊何归（唯一绑定决策）

终章 ch4 中段经 `gotoTrust` 进入"青囊何归"界面，玩家做出**唯一决定**：

- **托付于三人之一**：王济 / 陈伯 / 玄音道人（各对应一个结局）。
- **毁去残卷**：不托付于任何人，焚书（对应焚书结局）。

该决策直接写入 `finalChoice`，由 `resolveEnding` 解析为结局。决策完成后回到 ch4，华佗作出回应，进入结局。

### 6.5 结局（`ENDINGS`，4 个）

| 结局 id | 名称 | 分级 | 决策 | 墓志（节选） |
|---|---|---|---|---|
| chenbo_true | 医者人间 | 典故修补 | 交给陈伯 | 书没有留下，医道留下了。 |
| xuanyin_fallback | 残卷余音 | 遗憾半修 | 交给玄音道人 | 只有只言片语，流传于世。 |
| wangji_trap | 重锁深阁 | 遗憾未竟 | 交给王济 | 大部分内容，被锁进曹氏秘库。 |
| burn_ending | 青囊焚尽 | 遗憾焚绝 | 毁去残卷 | 外科之术，从此失传。 |

> 设计意图：把《青囊经》交给"真心行医、能让知识流入民间"的陈伯 = 最佳"修补"；其余皆为不同程度的遗憾。

## 7. 界面规格（页面清单）

路由以应用内 `page` 状态切换（非 URL）。`PageKey`：`cover / world / dungeon / story / clue / trust / map / ending / gallery`。

| 页面 | 组件 | 作用 | 主要交互 |
|---|---|---|---|
| 封面 Cover | CoverPage | 主视觉海报 + 行动入口 | 开始修补 → dungeon；查看设定 → world |
| 典籍设定 World | WorldPage | 世界观四卡 | 进入副本 → dungeon |
| 副本·典籍 Dungeon | ChapterSelectPage | 当前副本卡 + 未启副本（锁定） | 进入此卷 → story |
| 剧情 Story | StoryPage | 主叙事（场景 + 对白/旁白/抉择） | 轻触推进；抉择；底部导航 |
| 线索板 Clue | CluePage | 线索网格 + 详情抽屉 | 点开线索；回到医馆 |
| 青囊何归 Trust | TrustRoutePage | 终章唯一决策 | 选人 / 毁去残卷 → 确认 |
| 副本进程 Map | ProgressPage | 五章时间轴 + 展开卡 | 进入剧情 |
| 结局 Ending | EndingPage | 结局演绎（印章 + 余音） | 看图鉴 / 重新选择 / 青史空间 |
| 结局图鉴 Gallery | GalleryPage | 已解锁/未解锁结局 | — |

底部导航（仅剧情相关页）：**剧情 / 线索 / 进程 / 图鉴**。

## 9. 状态与存档

### 9.1 游戏状态 `GameState`（节选）
- `currentChapter`：当前章节号。
- `finalChoice`：终章决策（`wangji/chenbo/xuanyin/burn`），决定结局。
- `trustedPerson`：托付对象（焚书时为空）。
- `searchedClues`：已收集线索 id 列表。
- `unlockedEndings` / `lastEnding`：结局解锁记录。
- 章内 flag：`firstImpression / suspect / caoCunning / cao_suspicion / trust_huatuo / found_clue` 等（记录用，暂不参与结局判定）。

### 9.2 持久化
- **主存档**：localStorage 键 `loremender:huatuo:v2`，自动保存。
- **阅读位置**：localStorage 键 `loremender:huatuo:beat`，按章节标记，`{c: 章节, i: beat 下标}`；用于离开剧情页后**续读**；换章 / 新开局自动归零。
- 进入游戏时若已有存档则继续；"青史空间/重置"恢复默认；"重新选择"清空抉择并从头。

## 10. UI/UX 设计规范

### 10.1 视觉风格
- **基调**：暗夜古风。水墨、宣纸、烛火、金箔、朱印。
- **配色令牌（CSS 变量）**：
  - 墨：`--ink-deepest #0d0905` … `--ink-soft #3a2c1d`
  - 纸：`--paper #ebd9b3` / `--paper-bright #f3e5c2`
  - 金：`--gold-pale #e7c773` / `--gold #c9a14a` / `--gold-deep #8c6b29`
  - 朱：`--vermillion #a8311f` / `--vermillion-deep #6b1a10`
  - 烛：`--candle #ffb968`；夜：`--night #0f1620`
- **字体**：标题 `ZCOOL XiaoWei / Ma Shan Zheng`；正文 `Noto Serif SC`；英文 `Cormorant Garamond`。
- **纹理/特效**：颗粒 grain、暗角 vignette、金尘粒子 Particles、印章盖印动画 sealStamp。

### 10.2 版式与容器
- **移动优先**，逻辑分辨率以 390px 宽为基准。
- 视口高度使用 `100dvh`（含 `vh` 回退）；移动端锁定 `html/body/.stage` 外层滚动，仅内部 `.page-scroll` 滚动，避免地址栏收起导致整屏溢出。
- 桌面以居中"手机框"（`.phone`，圆角 + 刘海）预览；≤480px 时铺满全屏。

### 10.3 交互与无障碍
- 共享交互层：按钮/选项/图标按钮在桌面提供 `:hover` 反馈（`@media (hover:hover)` 隔离，触屏不粘连）。
- 键盘 `:focus-visible` 金色焦点环；统一 `:disabled` 态。
- 尊重 `prefers-reduced-motion`：降级动画。
- 图标按钮提供 `aria-label`；选择类卡片提供 `aria-pressed`。

### 10.4 关键交互细节
- **对白打字机**：逐字显示；点一次补全文本，再点推进；对白框**顶部对齐固定位置**，避免逐字增高引起的晃动；对白框非缩放（不带 `.press`）。
- **Toast**：限定在手机框内、居中、可换行（专用 keyframe 保留居中位移）。
- **托付卡片**：等高（`grid-auto-rows:1fr` + 卡片 `height:100%`）。

### 10.5 组件清单
`Topbar`（统一返回/标题/右槽）、`PageHeader`（眉标→标题→金线→引导语）、`BottomNav`、`SceneFrame`、`DialogueBox`、`ChoiceList`、`CharacterCard`、`PaperPanel`、`SealTag`、`GoldDivider`、`ProgressDots`、`BottomSheet`、`Toast`、`LockedDungeon`，以及 `art/` 下的场景/角色/图标 SVG。

## 11. 技术架构

| 项 | 选型 |
|---|---|
| 构建 | Vite 8 |
| 框架 | React 19 + TypeScript |
| 状态 | React `useState` 提升至 `App`，props 下传；localStorage 持久化 |
| 路由 | 应用内 `page` 状态（无 URL 路由）；`/editor` 路径加载编辑器 |
| 样式 | 全局 `global.css`（设计令牌 + 组件类）+ 内联样式 |
| 美术 | 内联 SVG（场景/角色/图标/封面）+ `public/images/cover.jpg` 主视觉 |
| 编辑器依赖 | `@xyflow/react`（可视化剧情流程图）|

### 11.1 目录概览
```
src/
  App.tsx                 路由与全局状态
  pages/                  各页面
  components/             共享组件（含 art/）
  data/
    types.ts              类型
    dungeons/huatuo/      华佗副本数据（story/characters/clues/endings）
    index.ts              统一导出
  lib/                    storage / routes
  styles/global.css       设计令牌与组件样式
  editor/                 可视化剧情编辑器（懒加载，/editor）
public/images/cover.jpg   首页主视觉
tools/twine-converter/    Twine 互转脚本
```

### 11.2 内容编辑器与 Twine
- **可视化剧情编辑器**（`src/editor/`，基于 `@xyflow/react`）：以节点图编辑剧情 beats / 跳转，路径 `/editor` 懒加载。
- **Twine 互转**（`tools/twine-converter/`）：剧情数据 ↔ Twine 格式导入导出，便于用 Twine 编写后回灌。
